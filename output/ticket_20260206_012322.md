# タイトル: 従業員情報に「趣味・嗜好」入力欄を追加

## 要約: 
従業員情報に「趣味・嗜好」を入力できる機能を追加し、人事部や総務部などが従業員の興味関心を把握しやすくすることで、福利厚生の企画立案の効率化を図る。現状、趣味・嗜好に関する情報が散在しており、企画担当が情報収集に時間がかかっている課題を解決する。

## ユーザーストーリー: 
- **[誰が]**: 人事部、総務部、福利厚生を担当する部署の担当者、および全従業員
- **[何のために]**: 福利厚生の企画立案を効率化するために
- **[何ができる]**: 従業員が自分の趣味・嗜好を登録し、管理側が一覧・検索できるようになる

## 開始条件: 
- [x] 従業員マスタの設計が完了していること
- [ ] 開発環境がセットアップされていること

## 完了条件: 
- [x] 従業員が自分の趣味・嗜好を登録できること
- [x] 管理側が従業員の趣味・嗜好を一覧・検索できること

## 実実装内容: 
> 修正・新規作成が必要なモジュールとロジックの要点

1. **`src/employee_master.py` の修正:**
    - **`add_employee()` 関数:**
        - 趣味・嗜好の入力を求めるプロンプトを追加する。例: `hobby = input('趣味・嗜好: ')`
        - CSVファイルへの書き込み処理に、趣味・嗜好の項目を追加する。`writer.writerow([emp_id, name, dept, org_id, hobby])`
    - **`list_employees()` 関数:**
        - CSVファイルから趣味・嗜好の項目を読み込み、一覧表示に追加する。
        - ヘッダー表示の変更: `print('ID\t名前\t部署\t所属組織ID\t趣味・嗜好')`
        - データ表示の変更: `print('\t'.join(row))` を `print('\t'.join(row[:4]) + '\t' + row[4])` に変更（rowが5要素以上の場合のみ）。または、より安全に `print('\t'.join(row))` を維持し、rowの要素数が期待通りでない場合の例外処理を追加。
    - **`EMPLOYEE_CSV` の修正:**
        - CSVファイルのヘッダー行を修正または新規作成し、趣味・嗜好の項目を追加する。既存データがある場合、データの移行が必要になる可能性がある。

2. **データ構造の変更:**
    - 従業員データを保存するCSVファイルの構造を変更する。具体的には、趣味・嗜好を格納するカラムを追加する。

3. **検索機能の実装 (オプション):**
    - 趣味・嗜好による従業員の検索機能を追加する場合、`employee_master.py`に検索関数を実装する。
    - 例: `search_employees_by_hobby(hobby)`

4. **UIの検討 (今回のコードにはUIは含まれていませんが、GUIやWebアプリケーションへの展開を考慮した場合):**
    - 趣味・嗜好の入力欄を設ける。
    - 一覧表示に趣味・嗜好の項目を追加する。
