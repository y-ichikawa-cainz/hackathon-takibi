# タイトル: 従業員情報画面に「趣味・嗜好」入力欄を追加

## 要約: 従業員情報に趣味・嗜好の入力欄を追加し、人事・総務部が福利厚生企画時に従業員の興味関心を把握しやすくする。趣味・嗜好に関する情報が散在している現状を改善し、企画担当の情報収集時間を削減する。

## ユーザーストーリー:
- **[誰が]**: 人事部、総務部、福利厚生を担当する部署、および全従業員
- **[何のために]**: 福利厚生の企画立案を効率化し、従業員のニーズに合った企画を実施するため
- **[何ができる]**: 従業員は自分の趣味・嗜好を登録でき、管理者は登録された趣味・嗜好の一覧を確認・検索できる

## 開始条件:
- [x] 従業員マスタ登録・編集画面が存在する
- [x] 従業員マスタのデータを保存するCSVファイルが存在する

## 完了条件:
- [x] 従業員情報登録時に「趣味・嗜好」を入力できる
- [x] 登録した「趣味・嗜好」が従業員情報一覧に表示される
- [x] 管理者が「趣味・嗜好」で従業員を検索できる

## 実装内容:
> 修正・新規作成が必要なモジュールとロジックの要点

1. **`src/employee_master.py` の修正:**
    - **`add_employee()` 関数:**
        - 趣味・嗜好を入力するための `input()` 関数を追加。
        - 入力された趣味・嗜好をCSVファイルに書き込む処理を追加。
    - **`list_employees()` 関数:**
        - CSVファイルから趣味・嗜好を読み込み、従業員情報一覧に表示する処理を追加。
        - ヘッダー行に「趣味・嗜好」カラムを追加。
    - **ファイル全体:**
        - CSVファイルのヘッダー定義に「趣味・嗜好」カラムを追加。
        - 検索機能実装のための準備（必須ではないが、今後の拡張性を考慮）。

2. **`src/employee_master.py` の変更点詳細:**
    - **`EMPLOYEE_CSV`**: CSVファイルの構造を変更し、趣味嗜好カラムを追加することを考慮する。
    - **`add_employee()`**:  `趣味・嗜好` の入力を求めるプロンプトを追加。取得した値を `writer.writerow()` に追加。
    - **`list_employees()`**: ヘッダー表示に `趣味・嗜好` を追加。各行のデータ表示にも `趣味・嗜好` を追加。
    - **検索機能**: (追加機能) 趣味・嗜好による検索オプションをメニューに追加し、検索処理を実装する。

3. **CSVファイル(`employees.csv`)の構造変更:**
    - 既存のCSVファイルに「趣味・嗜好」カラムを追加。

4. **UIの変更 (もしUIが存在する場合):**
    - 従業員情報登録画面に「趣味・嗜好」入力フィールドを追加。
    - 従業員情報一覧画面に「趣味・嗜好」カラムを追加。
